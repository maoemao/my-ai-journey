# 如何写好提示词

这是一个非常核心的问题，尤其是你现在在学习大语言模型的开发。写好提示词（Prompt）本质上就是**让AI「懂你在说什么、要它干什么、以什么方式干」**。

我给你系统讲一下思路，从浅到深分四层：

------



## **🧩 一、基础结构：提示词的骨架**



一个高质量提示词通常包含四个核心要素：

| **要素**                      | **作用**             | **示例**                                 |
| ----------------------------- | -------------------- | ---------------------------------------- |
| **角色设定（Role）**          | 告诉AI「你是谁」     | “你是一名资深全栈工程师。”               |
| **任务描述（Task）**          | 明确「要做什么」     | “请帮我写一个FastAPI的用户注册接口。”    |
| **输入信息（Context/Input）** | 提供必要背景或数据   | “用户表结构如下：…”                      |
| **输出要求（Output）**        | 限定格式、风格、语气 | “用markdown代码块输出，并附上简短解释。” |

👉 简单模板：

```
你是一名[角色]。
任务：[任务描述]。
输入：[上下文信息]。
输出要求：[格式/风格/语气等]。
```



------



## **🧠 二、表达原则：让模型「听得懂」**



提示词写得好，不是堆字，而是让AI能“稳稳抓住重点”：



1. **明确**：避免模糊动词（如“分析一下” → “列出3个优缺点，并总结结论”）。
2. **结构化**：分点描述，不要一整段堆成一句。
3. **约束输出**：指定格式、语气、风格（如“用JSON输出”，“语气正式”）。
4. **少歧义**：AI无法脑补你没写的上下文。
5. **示例驱动（Few-shot）**：给例子比光说更有效。



👉 例子（差 vs 好）：



**差：**



> 请帮我写一段python代码操作数据库。



**好：**



> 你是一名资深Python后端工程师。

> 任务：编写一段Python代码，用SQLAlchemy连接PostgreSQL数据库，并执行一个简单查询。

> 输出：



- > 使用async方式

- > 输出结果为一个包含用户名列表的JSON

- > 用Markdown代码块展示

------



## **⚙️ 三、进阶技巧：让AI「更听话」**



| **技巧**                         | **示例**                                         | **用途**         |
| -------------------------------- | ------------------------------------------------ | ---------------- |
| **链式思考（Chain of Thought）** | “先列出推理步骤，再给最终答案。”                 | 提高逻辑推理质量 |
| **逐步执行（Step-by-step）**     | “一步一步地讲解推理过程。”                       | 适合复杂任务     |
| **自我检查（Self-critique）**    | “完成后检查是否有语法错误或逻辑问题。”           | 提高稳定性       |
| **格式约束（Output schema）**    | “用JSON格式输出：{title:…, code:…}”              | 提高结构化结果   |
| **多角色协作（Role prompting）** | “现在你扮演产品经理，我扮演开发，我们讨论功能。” | 提升交互质量     |



------



## **🧰 四、开发者视角：提示工程（Prompt Engineering）**

如果你是开发AI应用，这几种模式要熟悉：



1. **Zero-shot**：直接问 → “写一首五言诗”
2. **Few-shot**：加例子 → “以下是两首例子，请再写一首”
3. **Chain-of-Thought**：引导模型先思考再回答
4. **ReAct**：让模型边推理边行动（Agent模式）
5. **Tool-use Prompt**：告诉模型如何调用外部工具（在LangChain、Dify等框架中很常见）

------



## **🧩 五、实战建议**

1. **先从结果出发**：先想清楚“我希望AI输出什么样的内容”，再写Prompt。
2. **多迭代调试**：优秀的Prompt不是一次写成，是多次试验调整。
3. **保留好版本**：每次调出满意结果，保存Prompt模板。
4. **模块化思维**：拆分为可复用块，比如“角色设定模板”、“输出规范模板”。
5. **自动化优化**：后期可以用AI自我改写Prompt（Auto Prompting）提高效果。
